確率シミュレーターの仕組みと計算ロジック

このページでは、本サイトに実装されている確率シミュレーターの内部ロジックや仕組みをわかりやすく解説します。 数値やレアリティの変化がどのように確率へ反映されているかを知ることで、より深く楽しめるはずです。

1. シミュレーターの基本構造

シミュレーターでは、選択したモンスターやキャラクターの「価値（Value）」を元に、合計値を算出しています。 各モンスターには固有の Value（価値スコア） が設定されており、選択した5体の合計に応じて、出現確率やシークレット発生率が変動します。

例：
・合計が 500 以下 → Secret 出現率 約15%以下
・合計が 750〜1000 → Secret 出現率 75%、BrainrotGod 25%
・合計が 1000 以上 → Secret 確定（100%）

2. 色（レアリティ）ごとの確率補正

各画像に設定できる「色ボタン（Default / Gold / Diamond / Rainbow / Halloween / Other）」は、内部的に確率を補正しています。

  • Default：標準的な9%
  • Gold：ボーナス率が高め（約+10%）
  • Diamond：やや希少（+5%前後）
  • Rainbow / Halloween：特殊ボーナス（イベント時限定）

選択した画像群の合計値に応じて、各色の割合が自動で再計算され、確率バッジに反映されます。

3. 合計値と待機時間（Wait Time）

合計値によって、結果が反映されるまでの待機時間を擬似的に設定しています。これは、ゲーム内でのレアリティ演出や成長速度のシミュレーションを再現するためです。

  • 合計が 750 以下：(Wait 1h0m)
  • 合計が 751〜5000：(Wait 1h30m)
  • 合計が 5001 以上：(Wait 2h0m)

4. 表示上の工夫

画像選択エリア（Select欄）は、モンスター画像と確率調整ボタンをわかりやすく並べた構成になっています。 画像上部の黒帯に表示される数値（K/s）は、内部Valueを表しており、演出上は「生成速度」や「エネルギー効率」を意味します。

5. よくある質問（FAQ）

Q1. この確率は実際のゲーム内確率と同じですか？

いいえ。本シミュレーターの確率はあくまで参考値であり、ゲーム内の実際の抽選ロジックを再現したものではありません。

Q2. 数値を変更したい場合は？

本ツールはオープンソース構成に準じているため、JavaScriptファイル内の images[] 配列を編集すれば自由に調整できます。

Q3. 広告掲載や収益化は可能ですか？

Google AdSense等の利用を想定しています。サイトポリシーに違反しない限り、このページを情報コンテンツとして申請することで審査対象となります。
